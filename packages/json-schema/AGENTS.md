# JSON Schema Package Guide (extends root `AGENTS.md`)

This package is the locally maintained fork of `@formily/json-schema`. Keep runtime semantics and the schema protocol close to upstream unless Silver Formily intentionally documents a divergence.

## Source Layout

- `src/` mirrors the upstream package structure so future syncs stay mechanical.
- Public exports flow through `src/index.ts`; avoid relying on deep imports from generated output.
- `esm/` is generated by tsdown and should be treated as build output only.

## Package Rules

- Runtime dependencies should stay on workspace packages (`@silver-formily/core`, `@silver-formily/reactive`, `@silver-formily/shared`) instead of reaching back to upstream Formily packages.
- When patching schema compilation, reactions, or polyfills, preserve existing protocol behavior and prefer adding regression coverage in downstream consumers if no local tests exist yet.
- Keep compatibility-focused changes small and obvious so the package can keep tracking upstream `formily_next/packages/json-schema`.

## Verification

- Run `pnpm --filter @silver-formily/json-schema build` after source changes.
- Run `pnpm --filter @silver-formily/json-schema check-types` when touching public types or build config.
- Rebuild dependent workspaces such as `@silver-formily/vue`, `@silver-formily/element-plus`, and `@silver-formily/vant` after changing exported schema behavior.
